# 极端场景组合交互测试报告

## 测试依据：核心逻辑规则 (Ground Truth Rules)

### 规则定义
1. **邪恶双子 (Evil Twin) 规则**：
   - 如果 **Evil Twin** 存活 且 **Twin Pair (他对立面的好人)** 也存活，**好人阵营无法获胜**（即使恶魔死亡，游戏继续）。
   - 如果 **Twin Pair** 被处决，**邪恶阵营立即获胜**。

2. **女巫 (Witch) 规则**：
   - 女巫在夜晚诅咒一名玩家。如果该玩家在白天发起提名，该玩家立即死亡。

---

## 🧪 场景一：双子死锁 (The Twin Lock)

### 输入状态
- `Demon` (Imp): 已死亡 (`isDead=true`)
- `Evil Twin`: 存活
- `Twin Pair` (比如 Washerwoman): 存活
- 其他爪牙: 已死亡

### 待测函数
`checkGameOver(seats, evilTwinPair, executedPlayerId)`

### 预期输出
`null` (游戏继续)

### 代码路径分析

**代码位置**：`app/engine.ts` 第 155-170 行

```typescript
// 1.4 如果恶魔死亡，检查镜像双子是否阻止好人胜利
if (deadDemon && !aliveDemon) {
  if (evilTwinPair) {
    const evilTwin = seats.find(s => s.id === evilTwinPair.evilId);
    const goodTwin = seats.find(s => s.id === evilTwinPair.goodId);
    const bothAlive = evilTwin && !evilTwin.isDead && goodTwin && !goodTwin.isDead;
    
    // 检查 Evil Twin 是否中毒或醉酒（中毒/醉酒的角色会失去能力）
    const evilTwinDisabled = evilTwin && (evilTwin.isPoisoned || evilTwin.isDrunk);
    
    // 只有当双子都存活且 Evil Twin 未中毒/未醉酒时，才阻止好人胜利
    if (bothAlive && !evilTwinDisabled) {
      // 恶魔死亡但双子都存活且 Evil Twin 能力有效，善良无法获胜，游戏继续（返回 null）
      return null;
    }
  }

  // 恶魔死亡且没有双子阻止，好人胜利
  const isSuccessor = deadDemon.isDemonSuccessor;
  return {
    winResult: 'good',
    winReason: isSuccessor ? '小恶魔（传）死亡' : '小恶魔死亡',
  };
}
```

### 验证结果

✅ **逻辑正确**

**执行流程**：
1. 第 156 行：检查 `deadDemon && !aliveDemon` → ✅ 满足（恶魔死亡，没有活着的恶魔）
2. 第 157 行：检查 `evilTwinPair` → ✅ 满足（存在双子配对）
3. 第 158-159 行：找到 Evil Twin 和 Good Twin
4. 第 160 行：`bothAlive = true`（两个都存活）
5. 第 163 行：`evilTwinDisabled = false`（假设 Evil Twin 未中毒/未醉酒）
6. 第 166 行：`bothAlive && !evilTwinDisabled = true`
7. 第 168 行：返回 `null`（游戏继续）✅

**检查重点验证**：
- ✅ 在判定 Demon 死亡导致好人胜利（第 172-177 行）**之前**，先插入了 `if (bothAlive && !evilTwinDisabled) return null` 的拦截逻辑（第 166-168 行）
- ✅ 拦截逻辑正确：只有当双子都存活且 Evil Twin 未中毒/未醉酒时，才阻止好人胜利

## 🧪 场景二：苦肉计胜利 (Win by Execution)

### 输入状态
- `Demon`: 存活
- `Twin Pair` (好人): 被提名并被处决 (`isExecuted=true`, `isDead=true`)

### 待测函数
`checkGameOver(seats, evilTwinPair, executedPlayerId)`

### 预期输出
`'evil'` (邪恶阵营胜利)

### 代码路径分析

**代码位置**：`app/engine.ts` 第 124-133 行

```typescript
// 1.1 如果善良双子被处决，邪恶阵营立即获胜
if (executedPlayerId !== null && evilTwinPair) {
  const executedPlayer = seats.find(s => s.id === executedPlayerId);
  if (executedPlayer && executedPlayer.id === evilTwinPair.goodId) {
    return {
      winResult: 'evil',
      winReason: '镜像双子：善良双子被处决',
    };
  }
}
```

### 验证结果

✅ **逻辑正确**

**执行流程**：
1. 第 125 行：检查 `executedPlayerId !== null && evilTwinPair` → ✅ 满足（有被处决的玩家且存在双子配对）
2. 第 126 行：找到被处决的玩家
3. 第 127 行：检查 `executedPlayer.id === evilTwinPair.goodId` → ✅ 满足（被处决的是 Good Twin）
4. 第 128-131 行：返回邪恶胜利 ✅

**检查重点验证**：
- ✅ 在检查恶魔死亡（第 135-178 行）**之前**，优先检查了双子对立面被处决的胜利条件（第 124-133 行）
- ✅ 优先级正确：Good Twin 被处决的判定在恶魔死亡判定之前

## 🧪 场景三：女巫的诅咒 (The Witch's Curse)

### 输入状态
- `Player A` 在昨晚被女巫选中（标记为 `witchCursedId`）
- `Player A` 发起提名（触发 `executeNomination`）

### 待测函数
`executeNomination(sourceId, id, options)` (在 `app/page.tsx` 中)

### 预期行为
`Player A` 立即死亡

### 代码路径分析

**代码位置**：`app/page.tsx` 第 5823-5833 行

```typescript
// 女巫：若被诅咒者发起提名且仍有超过3名存活，则其立即死亡
if (witchActive && witchCursedId !== null) {
  const aliveCount = seats.filter(s => !s.isDead).length;
  if (aliveCount > 3 && witchCursedId === sourceId) {
    addLog(`${sourceId+1}号 发起提名，触发女巫诅咒，立刻死亡`);
    killPlayer(sourceId, { skipGameOverCheck: false, recordNightDeath: false });
    setWitchCursedId(null);
    setWitchActive(false);
    return;
  }
}
```

### 验证结果

✅ **逻辑正确**

**执行流程**：
1. 第 5824 行：检查 `witchActive && witchCursedId !== null` → ✅ 满足（女巫激活且存在被诅咒的玩家）
2. 第 5825 行：计算存活玩家数量
3. 第 5826 行：检查 `aliveCount > 3 && witchCursedId === sourceId` → ✅ 满足（存活玩家 > 3 且提名者是被诅咒的玩家）
4. 第 5827 行：记录日志
5. 第 5828 行：调用 `killPlayer(sourceId, ...)` 立即杀死提名者 ✅
6. 第 5829-5830 行：清除诅咒状态
7. 第 5831 行：返回（阻止后续提名逻辑）

**检查重点验证**：
- ✅ 逻辑包含：`if (witchActive && witchCursedId === sourceId) { killPlayer(sourceId); }`
- ✅ 额外检查：存活玩家数量 > 3（符合女巫规则：如果只有三名存活的玩家，女巫失去此能力）
- ✅ 执行顺序：在设置提名记录（第 5834 行）之前检查，确保被诅咒者无法完成提名
