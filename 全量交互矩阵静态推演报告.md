## 概述

本报告检查了所有信息角色的 Vortox 兼容性。Vortox（涡流/沃托克斯）的能力是：**镇民玩家的能力都会产生错误信息**。当 Vortox 存活时，所有信息角色的输出必须为假信息。

## 核心机制检查

### 统一的 Vortox 检测机制

代码中使用了统一的 Vortox 检测机制：

1. **引擎层**：`app/engine.ts` 提供了 `checkVortoxEffect()` 和 `checkVortoxActive()` 函数
```typescript
   export function checkVortoxEffect(seats: Seat[]): boolean {
     const aliveVortox = seats.find(s => 
       !s.isDead && 
       (s.role?.id === 'vortox' || (s.isDemonSuccessor && s.role?.id === 'vortox'))
     );
     return !!aliveVortox;
   }
   ```

2. **UI 层状态**：`page.tsx` 中维护了 `isVortoxWorld` 状态，通过 `checkVortoxActive(seats)` 自动更新

3. **假信息判断**：所有信息角色通过 `shouldShowFake` 变量统一判断：
```typescript
   const fakeInfoCheck = drunkFirstInfoMap 
     ? shouldShowFakeInfo(targetSeat, drunkFirstInfoMap, vortoxWorld)
     : { showFake: isPoisoned || !!vortoxWorld, isFirstTime: false };
   const shouldShowFake = fakeInfoCheck.showFake;
   ```

## 关键发现

### 1. 统一的假信息生成机制

代码中有一个 `getMisinformation` 对象，目前包含两个辅助函数：
- `getMisinformation.fortuneTeller(realResult: boolean)`: 为占卜师生成假结果
- `getMisinformation.empath(realCount: number)`: 为共情者生成假数字（0/1/2）

**建议**：可以考虑扩展这个机制，为其他需要生成假信息的角色添加类似的辅助函数。

### 2. Empath 的假信息生成逻辑

**已验证**：Empath 测到 1 个邪恶邻居时，系统通过 `getMisinformation.empath(1)` 生成假数字。该函数会从 `[0, 1, 2]` 中排除真实值 `1`，从剩余值 `[0, 2]` 中随机选择一个，**完全符合 Vortox 规则**（必须给假信息，不能给真信息）。

### 3. 需要立即修复的问题

1. **Ravenkeeper (守鸦人)**: 在 `page.tsx:4129` 的查验逻辑中，只检查了 `currentHint.isPoisoned`，未检查 `isVortoxWorld`。需要修改为：
```typescript
   if (!currentHint.isPoisoned && !isVortoxWorld) {
     // 健康状态：显示真实身份
   } else {
     // 中毒/醉酒/Vortox 状态：弹出选择假身份的弹窗
     setShowRavenkeeperFakeModal(newT[0]);
   }
   ```

2. **Mathematician (数学家)**: `page.tsx:954-963` 中 `failCount` 始终为 0，这是因为数学家的能力需要追踪整个夜晚的行动历史（哪些能力被保护/免疫等阻止），目前未实现完整的追踪逻辑。但 Vortox 假信息生成逻辑本身是正确的（当 `shouldShowFake` 为 true 时会生成假数字）。建议后续实现完整的 `failCount` 计算。
